Strategy learnt from this application: having 3 main module of code to do things

- 1st module: Data handler.
This module contain: 
+, Having an object that stores all the data 
+, Methods for adding, deleting and storing data
+, Methods for input handling for data output

- 2nd module for UI controller
This module contains: 
+, String for classes in html
+, Method for adding / deleting html script 
+, String representation manipulation (for example: number formatting)
+, etc ...

- 3rd module for global controller 
This module contains: 
+, Event listener
+, Code to apply all the method for both the module above
